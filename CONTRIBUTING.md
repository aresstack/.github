# Contributing to aresstack

Thanks for your interest in contributing.

## Scope

aresstack maintains open-source Java libraries intended for production use and Maven Central distribution.

## Before You Start

- Open an issue for bug reports and feature ideas before large changes.
- Keep pull requests focused and small.
- Prefer backward-compatible changes unless a breaking change is clearly justified.

## Development Expectations

- Follow repository coding style and existing architecture.
- Add or update tests when behavior changes.
- Update documentation when public APIs, behavior, or configuration changes.
- Consider Java version compatibility and transitive dependency impact.

## Pull Requests

When submitting a PR:

1. Explain the problem and the approach.
2. Reference related issues.
3. Include tests or clear validation steps.
4. Highlight any compatibility, migration, or release concerns.

Maintainers may request revisions to ensure maintainability, release quality, and long-term support.
