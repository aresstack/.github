# Security Policy

## Supported Projects

Security support depends on each repository's maintenance status and release lifecycle. Check the repository documentation and latest release notes for current support details.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately.

- Do **not** open public issues for security reports.
- Contact maintainers through GitHub Security Advisories (preferred) or repository security contacts where available.
- Include reproduction details, affected versions, and impact.

We aim to acknowledge reports promptly and provide updates as triage progresses.

## Responsible Disclosure

We ask reporters to keep details confidential until a fix is available and users have had a reasonable chance to update.
